About the Cartesi Bug Bounty

Cartesi is offering incentives for any problems encountered and fixed while running Cartesi Core on the Rinkeby testnet. This channel is being opened to provide a pathway to report security vulnerabilities that may encompass Cartesi Core or the decentralized tournament infrastructure.

This incentivized bug campaign will help us fine-tune our technology and prepare for our mainnet release down the road.

Any participants who are able to find a bug, vulnerability, or problem while running Cartesi Core or the decentralized tournament infrastructure can report it as an issue. The team will review all submissions and grant award payouts for critical vulnerabilities.

How to Report

If you would like to report a bug, vulnerability, or any other problem that you have discovered while running Cartesi Core and the decentralized tournament infrastructure surrounding Creepts, your submission must include the following items:

Eligible Vulnerability

The vulnerability must be within the reporting eligible scope for a potentially award payout.

Clear Writing

A clear title and a concise description summarizing the type of vulnerability and the impacted asset.

Steps to Reproduce

You must add as much details as possible for how we can reproduce the issue.

Supporting Material

List all additional material (e.g. screenshots, logs, etc.) that can help us quickly identify the vulnerability.

Report a bug

Rules

  • Vulnerability reports occur on a first report basis. Reports that are the same, after the first report, will not be granted an award.
  • Submissions should contain clear step-by-step instructions for Cartesiâ€™s Technical team to be able to reproduce it. The more detailed reports are, the higher the award payouts can potentially be.
  • A series of vulnerabilities stemming from one particular issue will count as one vulnerability.
  • Any issues made public will not be awarded a payout
  • Vulnerabilities already known or submitted by the team, will not be issued a reward

Scope

The following represents the scope in which issue reporting is eligible for an award payout.

  • cartesi-arbitration
  • cartesi-dispatcher
  • cartesi-logger
  • cartesi-machine-emulator
  • cartesi-machine-solidity
  • cartesi-machine-manager
  • creepts-tournament

Rewards

Bounty reward payouts are determined based on: the severity, report outline and based on risk-assessments conducted by the Cartesi team. The severity rates include: critical, high, medium and low vulnerabilities.

  • Low $100 in CTSI
  • Medium $500 in CTSI
  • High $1,000 in CTSI
  • Critical $2,000 in CTSI
